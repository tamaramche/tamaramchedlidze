Algorithms for Drawing Graphs with Constraints

Drawing constraints are restrictions on a graph visualization which may stem from an interaction process or an application semantics. When interacting with a graph visualization environment, the user may want to restrict positions of the vertices and the shape of the edges, but still use a specific drawing algorithm.

Monotone Drawings

Path following is one of the basic tasks that a user performs with a graph drawing. In several user studies, including eye-tracking (Huang 2007) and controlled experiments (Huang et al 2009), it was confirmed that people have geodesic-path tendency, that is they tend to follow edges that lead in the direction of the target vertex. Inspired by this finding, in 2010, Angelini et al. defined a new paradigm for graph visualization. A drawing of a graph is monotone if for every pair of vertices u and v there is a path from u to v such that the drawing of path implied by the drawing is monotone.

Point Set Embeddings of Planar Graphs

Point Set Embedding is a classical problem in the intersection of graph theory and computational geometry. Given a planar graph G=(V,E) and a set of points in the plane S, with |V|=|S|, the problem asks whether there exists a bijective mapping from V to S, such that the implied straight-line drawing of G is planar. The Point Set Embedding problem is rather of theoretical interest and does not seem to be motivated by a real-world setting.
